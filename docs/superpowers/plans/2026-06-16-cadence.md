# CADENCE Implementation Plan (All 7 Phases)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> **On execution, copy this file to** `cadence-inteligence/docs/superpowers/plans/2026-06-16-cadence.md` (plan mode only allowed editing the scratch plan file).

**Goal:** Build CADENCE — an autonomous content-intelligence platform for music artists/labels where a persistent agent watches catalog momentum and emits ready-to-use multiformat Content Packages — with the Musixmatch Pro API as the intelligence core.

**Architecture:** Next.js 16 App Router app (Supabase SSR auth + Postgres for *derived* data only) exposes a six-layer pipeline (SIGNAL → INTELLIGENCE → GENERATION → ASSET → COLLAB) as typed adapters + route handlers. An n8n agent loop (WATCH→DETECT→ANALYZE→GENERATE→PACKAGE→SURFACE) calls these endpoints on a schedule, writes to a live `agent_log`, and deposits Content Packages in the user's inbox. A hard compliance rule forbids persisting raw Musixmatch lyric content anywhere.

**Tech Stack:** Next.js 16 (App Router) + TypeScript (strict) + Tailwind 4 + Supabase (`@supabase/ssr`, Postgres + Storage) + Vercel AI SDK via AI Gateway (Anthropic Claude) + n8n. Partners: Musixmatch Pro, Songstats, Cyanite, ElevenLabs, LALAL.AI, JamBase. Tests: vitest (unit) + Playwright (e2e). Deploy: Vercel + hosted n8n.

## Global Conventions (apply to every task; do not repeat boilerplate)

**Reference files to copy patterns from:**
- Supabase clients: `cultural-map/lib/supabase/{client.ts,server.ts,middleware.ts}` — `createClient()` (anon, cookie-bound) + `createServiceClient()` (service role, RLS bypass).
- Middleware: `cultural-map/proxy.ts` (Next 16 names the entry `proxy.ts`, exports `proxy` + `config` matcher) calling `updateSession`.
- Auth: `cultural-map/app/auth/{login/page.tsx,actions.ts,callback/route.ts,signout/route.ts}` + `lib/auth/allowlist.ts` (magic-link OTP + email allowlist).
- Route handler shape: success `NextResponse.json({ data })`, failure `NextResponse.json({ error }, { status })`. Cron/agent endpoints gate on `authorization === \`Bearer ${process.env.CRON_SECRET}\``, set `export const runtime = "nodejs"` and `export const maxDuration = 60`.
- External API adapter shape (from `mw3-club/src/lib/fal.ts`): one module per partner under `lib/partners/<name>.ts`; read credentials from `process.env` inside the function (never at module top); typed input/output interfaces; zod-parse responses; throw on missing/invalid data; `fetchWithTimeout` (5s) helper.
- Tailwind 4: `app/globals.css` uses `@import "tailwindcss"` + `@theme inline` + OKLCh CSS variables + `.dark`; PostCSS via `@tailwindcss/postcss`; no `tailwind.config.ts`.
- TS alias `@/*` → `./*`; strict mode. Path layout: `app/`, `components/` (`ui/` + domain), `lib/` (split by concern), `supabase/migrations/`, `tests/{unit,e2e}/`.
- Supabase types: generated to `lib/supabase/types.ts` via `npm run db:types`; type all clients with `<Database>`.

**Commit after every task.** Conventional commit prefixes (`feat:`, `test:`, `chore:`).

## Compliance guardrail (HARD RULE — read before any data/persistence task)

- DB stores **derived, non-lyric data only**: Songstats numbers, Cyanite tags/curves, derived labels (theme/mood/language), generated briefs/plans/config, public creator metrics.
- **Never persist raw Musixmatch lyric content.** Hook lines and lyric clips are fetched **live**, used as **snippets < 15 words** in displayed copy, and assets are generated **on-demand and ephemerally**.
- No migration may add a column intended to hold lyric text. A vitest guard (Task 0.4) scans migrations + adapter code for forbidden patterns and is part of CI.

```ts
// lib/compliance/lyrics.ts
export function wordCount(s: string): number {
  return s.trim().split(/\s+/).filter(Boolean).length;
}
/** Lyric-derived snippets may be DISPLAYED only if < 15 words. Never persist raw lyrics. */
export function assertSnippetAllowed(snippet: string): string {
  if (wordCount(snippet) >= 15) {
    throw new Error("Compliance: lyric snippet must be < 15 words");
  }
  return snippet;
}
```
